The guts of the http message parsing process. 

Call this function repeatedly with successive buffers of data. These successive buffers may represent one or more messages and a single buffer is permitted to hold the end of one message and the start of the successive message.

When a full message is complete the parser will call the \textquotesingle{}message\+\_\+complete\+\_\+handler\textquotesingle{}. This function should process the message and send the reply before returning. This will suspend the parser until the processing of the message is complete. And if necessary will continue with the next message

The \mbox{\hyperlink{http__message__parser_8h_aca384d2eeb6275888ce4046eaf28f0e7}{http\+\_\+message\+\_\+parser\+\_\+consume()}} function will only return if\+:


\begin{DoxyItemize}
\item it has consumed all the data. So read some more and give it to consume.
\item an error was encountered. Either a parse error, or an io error -\/ This should be treated as fatal and the connection closed.
\item in a situation where a messsage does not contain a message length value the parser may need to be given an End-\/\+Of\+\_\+\+File signal in the form of a call to http\+\_\+message\+\_\+parser\+\_\+consume with a zero length buffer. This need for this will be signalled by llhttp\+\_\+message\+\_\+needs\+\_\+eof(parser-\/$>$m\+\_\+llhttp\+\_\+ptr) returning true
\end{DoxyItemize}
